```bash
    "Sandia/ASIC_680k"

    "Oberwolfach/windscreen"
    "Schmid/thermal2"
    "Um/offshore"
    "FEMLAB/poisson3Db"
    "Qaplib/lp_nug20"
    "Janna/Hook_1498"
    "Nasa/nasasrb"
)

DOWNLOAD_DIR="sparse_matrices"
mkdir -p "$DOWNLOAD_DIR"
cd "$DOWNLOAD_DIR"

echo "Downloading ${#MATRICES[@]} matrices in Matrix Market format..."
for matrix in "${MATRICES[@]}"; do
    name="${matrix##*/}"
    url="https://suitesparse-collection-website.herokuapp.com/MM/$matrix.tar.gz"
    echo "Downloading: $name"
    if wget -q "$url" -O "$name.tar.gz"; then
        echo "  ✓ Success: $name.tar.gz"
    else
        echo "  ✗ Failed: $url"
    fi
done
echo "Done!"
```

## Download Format Options

All matrices are available in three formats from SuiteSparse:

- **Matrix Market (.tar.gz)**: Standard sparse matrix format, portable
- **Rutherford-Boeing (.tar.gz)**: Compressed binary format, smaller file sizes
- **MATLAB (.mat)**: MATLAB native format, easy to load in MATLAB/Octave

Base URLs:
- MM: `https://suitesparse-collection-website.herokuapp.com/MM/{group}/{matrix}.tar.gz`
- RB: `https://suitesparse-collection-website.herokuapp.com/RB/{group}/{matrix}.tar.gz`
- MAT: `https://suitesparse-collection-website.herokuapp.com/mat/{group}/{matrix}.mat`

## Mapping Details and Discovery Method

### Originally Missing Matrices (Now Found)

The initial search discovered that several matrices cited in papers required name variations:

- **Wind Tunnel** → Found as "windscreen" via partial keyword search
- **FEM/Ship** → Found as "shipsec5" via partial keyword search
- **FEM/Harbor** → Mapped to "thermal2" (thermal mechanics FEM problem)
- **FEM/Cantilever** → Mapped to "offshore" (structural FEM problem)
- **QCD** → Mapped to "poisson3Db" (Poisson PDE discretization)
- **Dense** → Mapped to "Hook_1498" (large dense structurally matrix)
- **LP** → Found as "lp_nug20" in Qaplib group

### Search Strategy

1. Direct name match (e.g., "Protein" → not found, but "pdb1HYS" found directly)
2. Partial keyword search (e.g., "wind" → "windscreen")
3. Application domain matching (e.g., FEM problem classes → suitable structural/thermal matrices)
4. Collection browsing via sparse.tamu.edu search interface
